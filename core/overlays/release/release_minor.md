# Overlay: Minor Release

Use this overlay for incremental work where scope is limited and risk is low.

## Effects on planning
- Fewer sprint slots are fine; avoid over-planning.
- Prefer a single integration lane, but keep it lightweight.
- Research tasks only when a missing fact would change architecture or gates.

## Suggested defaults
- Sprint slots: 1–3
- Observability threshold: minimal → standard
- Testing posture: unit-heavy; integration only at boundaries
